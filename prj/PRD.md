# 제품 요구사항 문서

## 제품

HWPX Editor는 사용자가 Obsidian vault를 떠나지 않고 한컴 HWP/HWPX 파일을 바로 열고, 만들고, 편집할 수 있게 해주는 플러그인입니다.

## 문제

HWP 파일은 한국의 공공, 학교, 법률, 업무 환경에서 자주 쓰입니다. Obsidian은 이런 파일을 저장할 수는 있지만, 기본 기능으로 내용을 미리 볼 수는 없습니다.

## 목표

- Obsidian 파일 탐색기에서 `.hwp`와 `.hwpx` 파일을 엽니다.
- 문서 페이지를 Obsidian 파일 뷰 안에 렌더링합니다.
- 기본 상태는 read-only 보기로 둡니다.
- 사용자가 편집 모드에 들어가 명시적으로 저장하기 전까지 원본 문서를 변경하지 않습니다.
- 일반 보기와 편집 중 원격 네트워크 의존성을 피합니다.

## 목표가 아닌 것

- 첫 단계에서 완전한 HWP 편집 기능을 제공하는 것.
- `rhwp`가 제공하는 수준을 넘어 한컴과 완전히 동일한 렌더링을 보장하는 것.
- 첫 단계에서 모바일을 지원하는 것.

## 사용자 흐름

1. 사용자가 vault 안의 `.hwp` 또는 `.hwpx` 파일을 클릭합니다.
2. Obsidian이 HWPX Editor 파일 뷰를 엽니다.
3. 플러그인이 파일을 바이너리 데이터로 읽습니다.
4. 플러그인이 `@rhwp/core`를 초기화합니다.
5. 페이지가 스크롤 가능한 read-only 문서 영역에 SVG로 렌더링됩니다.
6. 렌더링 실패 시 읽을 수 있는 오류 메시지를 표시합니다.

## 편집 흐름

1. 사용자가 문서를 read-only 모드로 엽니다.
2. 사용자가 툴바에서 편집 버튼을 누릅니다.
3. 플러그인이 번들된 `rhwp-studio` 편집기를 `@rhwp/editor` iframe으로 로드하고 현재 파일 바이트를 전달합니다.
4. 사용자가 내장 편집기에서 문서를 편집합니다.
5. 사용자가 저장 버튼을 누릅니다.
6. 플러그인이 편집기에서 HWP/HWPX 바이트를 내보내 같은 vault 파일에 다시 씁니다.

사용자가 파일을 전환하거나, 뷰를 닫거나, read-only 모드로 돌아가면서 편집 모드를 떠나려 하면 플러그인은 계속 편집, 버리기, 저장 세 가지 선택지를 제공합니다.

## 생성 흐름

1. 사용자가 Obsidian 파일 탐색기에서 파일 또는 폴더를 우클릭합니다.
2. 설정된 기본 새 파일 형식에 따라 `새 HWP` 또는 `새 HWPX`를 선택합니다.
3. 플러그인이 대상 폴더에 빈 파일을 만듭니다.
4. 한국어 이름은 `새 파일`, `새 파일 2`, `새 파일 3`을 사용하고, 영어 이름은 `Noname`, `Noname 1`, `Noname 2`를 사용합니다.
5. 새 파일은 Obsidian에서 편집 모드로 열립니다.

## 인수 기준

- `npm run build`로 플러그인이 빌드됩니다.
- Obsidian이 생성된 `main.js`, `manifest.json`, `styles.css`를 로드합니다.
- `.hwp`와 `.hwpx` 파일이 플러그인 뷰에 연결됩니다.
- 문서를 여는 것만으로는 원본 파일에 다시 쓰지 않습니다.
- 뷰 상단에는 페이지 수와 새로고침 컨트롤이 표시됩니다.
- 편집 모드는 명시적인 툴바 동작이 있어야 진입합니다.
- 편집 모드는 명시적인 저장 동작 후에만 파일에 씁니다.
- 편집 모드를 떠날 때 저장 또는 버리기를 확인합니다.
- 파일 탐색기 컨텍스트 메뉴에서 충돌 없는 한국어 이름으로 빈 HWP/HWPX 파일을 만들 수 있습니다.
- 플러그인 UI와 알림은 Obsidian 언어 설정에 따라 한국어와 영어를 지원합니다.
- 툴바에는 설치된 rhwp core 버전이 표시됩니다.
- 설정된 기준보다 큰 파일을 열기 전에 확인할 수 있습니다.
- 설정에는 기본 새 파일 형식, 큰 파일 처리 방식, 큰 파일 기준 용량이 있습니다.
- 뷰에는 Obsidian 파일 통계에서 가져온 생성/수정 시각을 보여주는 접을 수 있는 속성 패널이 있습니다.

## 편집 모드 제약

`@rhwp/editor`는 iframe 기반이며 기본값으로 `https://edwardkim.github.io/rhwp/`의 rhwp-studio를 로드합니다. 이 플러그인은 `studioUrl`을 번들된 로컬 `rhwp-studio/index.html`로 덮어써서 read-only 모드와 편집 모드 모두 플러그인 로컬 자산을 사용합니다.
